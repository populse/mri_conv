Software - Bruker - Dicom - Philips - Nifti

Software

What Problem Possible cause Correction
Software The software does not start when double-clicking on 'MRIManager.jar' 1) your java is old or not installed
2) out of memory 		1) see install/update instructions
2) use MRIManager.bat (Windows) or MRIManager.sh (Linux, MAC) - see 'Installation' page
Software The software does not start when launching 'MRIManager.bat' or 'MRIManager.sh' 1) the maximum memory allocation pool is too high
2) under Linux, MRIManager.sh is not executable 		1) edit the MRIManager.bat or MRIManager.sh file according to your operating system, decrease the value of -Xmx (for example 4096m to 2048m)
2) Command line : just run 'chmod +x /path/to/your/MRIFileManager/MRIManager.sh'
Software An error message appears 'Warning: DictionaryMRI_System.yml not found' 1) the DictionaryMRI_System.yml and DictionaryMRI_User.yml files are corrupted or missing in the software directory 1) re-download MRI FIle Manager - see 'Installation' page

Bruker

What Problem Possible cause Correction
Bruker - data No data found in the table at the top (data browser) 1) the data folder selected is not right
2) the data format chosen is not right
3) the Bruker format is not compatible with the software
4) the 'subject' file is corrupted or missing		 1) select the directory containing the 'subject' file
2) click on the right button 'Bruker'
3) verify that Bruker format is Paravision 5 or Paravision 6
4) In the Bruker data folder, verify that the 'subject' file is present or not empty
Bruker - sequence No sequence found in the table at the bottom (MRI sequence browser)
(or a few sequences are missing) 		1) some Bruker metadata are missing
2) the '2dseq' file is missing or empty 		1) In the Bruker data folder, verify that the 'acqp', 'reco', 'method' and 'visu_pars' file are present or not empty
2) In the Bruker data folder, verify that the '2dseq' is present or not empty
Bruker - MRI parameters Some MRI parameters are missing in the 'MRI parameters' table
 1) some Bruker metadata are missing or empty
2) some informations in the DictionaryMRI_System.yml or DictionaryMRI_User.yml file are missing or improperly completed 		1) In the Bruker data folder, verify that the 'acqp', 'reco', 'method' and 'visu_pars' file are present or not empty
2) see 'MRI Dictionaries' page
